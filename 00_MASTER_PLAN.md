## 📚 Git Best Practices (Learning as You Go)

### Repo Structure
```
project-name/
├── README.md          # Project overview, setup instructions
├── docs/              # Additional documentation
├── src/               # Source code
├── tests/             # Test files
├── examples/          # Usage examples
└── .gitignore        # Ignore node_modules, venv, etc.
```

### Commit Message Format
```
type: brief description

- Detailed point 1
- Detailed point 2

Example:
feat: add ticket classification endpoint

- Implement Claude API integration
- Add input validation
- Include error handling
```

**Types:** feat, fix, docs, refactor, test, chore

### Branching Strategy
- `main` - production-ready code
- `develop` - work-in-progress
- `feature/feature-name` - specific features

### Weekly Git Checklist
- [ ] Meaningful commit messages
- [ ] Updated README
- [ ] Code commented
- [ ] No sensitive data committed
- [ ] .gitignore configured

## 🆘 What to Do When You're Stuck

### If You're Losing Interest (Week 2 Slump)
1. **Skip ahead to building** - Start a project before finishing theory
2. **Shrink the goal** - 20 min instead of 45 min for a few days
3. **Change the format** - Watch videos instead of reading, or vice versa
4. **Phone a friend** - Message me: "Feeling stuck, need motivation"

### If You're Behind Schedule
1. **Don't panic** - Progress > perfection
2. **Adjust timeline** - Extend by 1-2 weeks, not months
3. **Skip optional content** - Focus on core courses + projects
4. **Ask for help** - I'll help you re-prioritize

### If Technical Blockers Happen
1. **Document the blocker** - Write it down
2. **Set a 30-min timer** - Try to solve it
3. **Ask Claude/me** - Share the error, I'll help debug
4. **Move on if stuck** - Come back later, don't let one issue derail everything
